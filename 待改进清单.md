# BFHC 待改进清单

## 1. 退蚀全过程温度约束检查

当前圆柱簇和 zigzag 簇的共同求解思路是：先对初始几何搜索满足温度约束的最高工作电压 `Vwork`，随后在该固定电压下进入退蚀/侵蚀生命周期仿真。

需要注意的是，初始几何满足 `Tmax < 3273.15 K` 并不严格保证退蚀过程中所有时刻都满足温度约束。随着材料损失，局部截面积、电流密度、焦耳热密度、导热路径和辐射面积都会变化，细颈、转角或连接区域可能在某个退蚀步出现新的局部热点。

因此，后续应将温度约束从“初始工作点检查”扩展为“生命周期全过程检查”：

```text
对每个退蚀步重新求解温度场；
若任一步 curTmax >= 3273.15 K，则该结构判为不满足温度约束。
```

建议在 runner/Java 脚本中增加显式状态，例如：

```text
FAIL_OVERTEMP_DURING_EROSION
```

并在 CSV 中记录发生超温的退蚀步、累计时间、`curTmax` 和对应几何参数。这样可以避免某些初始可行但生命周期中后期超温的结构被误判为可行解，也能让技术报告中的约束口径更严谨。

## 2. selfViewLoss 绝对值偏差说明

当前均匀圆柱基线出现约 $-5.66\%$ 的 `selfViewLoss`，主要来自局部辐射通量积分与 S2S radiosity 网络之间的差异。具体而言，局部表面积分口径使用近似净辐射通量表达式，而外接球统计口径使用能量守恒路径估计对外净辐射功率，两者在 COMSOL 的 S2S 面-面辐射网络中并非完全等价。

由于所有候选结构都采用同一统计口径，该偏差主要影响 `selfViewLoss` 的绝对数值，不影响形状间的相对比较。因此当前优化仍可使用同一口径下的 `lifeAvgP03sphere_W`、寿命和相对提升作为主要评价依据，但在提交报告中需要明确说明该系统性偏差，避免将负值 `selfViewLoss` 解释为真实物理意义上的“负遮挡”。

## 3. Zigzag 退蚀温度统计口径升级

当前 Zigzag 簇的退蚀温度统计仍采用轻量化抛物线近似：先读取整体 `Tmax/Tmin`，再按每个折线块中心高度估算块平均温度。这一做法计算成本低，适合快速完成长路径结构的大规模搜索，但它没有像圆柱簇那样直接从 COMSOL 侧面温度积分中读取每段真实面积加权平均温度。

后续应将 Zigzag 温度统计升级为逐块侧面面积积分口径：

```text
为每个 Zigzag block 建立侧面选择集；
计算 TintBlock_i = ∫ T dA；
计算 ABlock_i = ∫ 1 dA；
使用 Tavg_i = TintBlock_i / ABlock_i 驱动该块退蚀。
```

这样可以减少抛物线近似对局部热点、转角温度和不同折返段位置的误差影响，使 Zigzag 寿命评估与圆柱簇当前提交版的温度统计口径保持一致。

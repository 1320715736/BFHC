## 1. 总结论

1. **当前代码不是完整符合复赛要求的最终物理模型。** 电流、焦耳热、导热、双波段发射率、表面对表面辐射、准稳态退蚀、20% 特征尺度失效和全过程超温检查已经具备，但外接球逃逸辐射、真实局部退蚀、瞬态温度和最终候选应力仍未闭环。
2. **最高优先级问题是外接球统计。** 两个 runner 都直接令 `P03sphere = P03`、`PradSphere = Prad`。当前字段名虽然叫 `sphere`，实际仍是自由表面的局部发射积分，不是穿过 0 K、发射率 1 外接球的净逃逸功率。
3. **当前 `selfViewLoss_pct` 基本恒为 0 是代码恒等式，不是自遮挡很小。** 表面对表面辐射会影响温度场，但没有被正确转换成评分所需的外部逃逸辐射，因此 zigzag 的遮挡惩罚没有进入 `E`。
4. **退蚀几何仍有明显近似。** 圆柱簇的台阶肩面参与辐射却没有对应法向退蚀；zigzag 虽保存逐块边长，却用最小边长统一重建所有块，端部圆柱 stub 和转折端面也未按真实局部表面退蚀。
5. **C10 已经测到当前模型下的真实 20% 失效，而不是 1000 h 截断。** 其寿命为 `647.0447 h`，但 `E=4.4405e9 J`、`selfViewLoss=0` 和候选排名不能作为最终比赛结论；寿命也要在修正 zigzag 局部退蚀后重算。
6. **现在不应继续扩大贝叶斯搜索。** C1-C10 到此作为历史阶段封存；下一阶段从 D 开始，先完成 D1-D6 的物理和数值门禁，再生成新版本 B2/B3 基线并重新搜索。旧数据保留作回归样本，不继续用于最终评分。

### 3.2 当前模型架构

- 圆柱簇：8 个轴向圆柱段参数化半径，COMSOL 稳态 `Electric Currents + Heat Transfer + Surface-to-Surface Radiation`，Python 外循环执行准稳态退蚀。
- Zigzag 簇：Manhattan 方截面 block 与两端圆柱 stub 的 Union，物理接口同上，Python 外循环保存逐块退蚀状态。
- 力学：只存在于 `*_process_views.java`，且仍是旧几何、固定参数和 4 个稳态展示阶段，未进入当前 C10 候选验证。
- 优化：Optuna 单目标搜索，目标主要由 `eta_E` 构成，默认额定电压策略为 `max_safe`，不是已有但未启用的全电压生命周期扫描。

## 5. 外接球问题的正确修复定义

### 5.1 当前错误不是“没有画一个球”这么简单

赛题中的外接球是三维**外接球统计面**，不是外接圆，也不是需要参与导热的实体。当前局部表达式

```text
eps03 * sigma * f03(T) * T^4
```

只能得到每个自由表面的 0 K 本征发射量。凹面、相对表面和密集 zigzag 中，一部分辐射会打到自身其他表面并被吸收或反射；只有最终逃逸到外部黑体环境的部分可以记为 `P03sphere`。

COMSOL 的 S2S 接口本身使用 radiosity 和 view factor，环境视角因子表示未被其他辐射面覆盖、能够逃逸到环境的比例。可依据官方 [Surface-to-Surface Radiation 接口](https://doc.comsol.com/6.3/doc/com.comsol.help.heat/heat_ug_interfaces.08.49.html)、[Radiosity Method](https://doc.comsol.com/6.3/doc/com.comsol.help.heat/heat_ug_theory.07.051.html) 和 [View Factor Evaluation](https://doc.comsol.com/6.3/doc/com.comsol.help.heat/heat_ug_theory.07.053.html) 实现；官方[同心球 View Factor 示例](https://doc.comsol.com/6.4/doc/com.comsol.help.models.heat.view_factor/view_factor.html)可作为回归基准。

### 5.2 推荐实现

1. 保留现有 `radThermal`：环境温度 293.15 K，参与稳态/瞬态热平衡。
2. 新建独立 `radScore` 或独立后处理研究：导入已经求得的钨温度场，环境/接收球设为 0 K、发射率 1，不反向改变热解。
3. 生产统计可用 S2S 环境逃逸 radiosity 与 `Famb` 积分；同时建立一个显式黑色球面接收验证模型，证明两种方法等价。
4. 分别输出：

```text
P03gross_W                 自由表面 0-3 um 本征发射总量
P03escape_W                最终逃逸到 0 K 黑色外部环境的净功率
P03selfAbsorbed_W          P03gross_W - P03escape_W
selfViewLoss_pct           P03selfAbsorbed_W / P03gross_W * 100%
lifeAvgP03escape_W         生命周期平均逃逸功率
lifeTotalP03escape_J       生命周期累计逃逸能量
```

5. 原字段 `P03sphere` 只可作为 `P03escape` 的兼容别名，必须同时写入 `metricVersion`，防止与旧 CSV 混用。

### 5.3 D1 验收测试

- 凸均匀圆柱的遮挡损失接近 0，且绝不因恒等赋值被强制为 0。
- 两个相向平板/折返结构的 `P03escape < P03gross`，增加遮挡后损失单调增大。
- 显式球面接收法与环境逃逸法差异 `<1%`。
- 球半径取包围半径的 1.05、1.5、2.0 倍时，接收总功率变化 `<0.5%`。
- 全波段能量平衡满足 `P_electric = P_end_conduction + P_radiation + P_sublimation`；未耦合潜热时应单列该项，主残差目标 `<1%`、最低接受 `<2%`。
- 任何正式结果若 radiosity 变量、选择集或球面积分读取失败，必须返回失败状态，禁止回退为局部发射量。

## 7. 还需修正的代码级问题

### 7.1 两个 runner 共性问题

- 正式统计失败时存在 `Tmin=0.95*Tmax` 等回退；这会生成看似完整但错误的 U。
- 结果没有 `physicsVersion/metricVersion/geometryVersion/COMSOLVersion/gitCommit/randomSeed/meshLevel`。
- 没有正式的网格、退蚀时间步和电压搜索收敛报告。
- COMSOL server 断连恢复机制已改善，但失败点、重试次数和旧失败行仍需统一状态机。
- 没有自动导出每个退蚀阶段的质量、几何、温度、功率和能量守恒历史。
- 热求解包含 3 um 外辐射，这是正确的；但字段命名应防止把“总热辐射”与“0-3 um 评分辐射”混淆。

### 7.2 圆柱簇特有问题

- 8 段阶梯圆柱并非光滑、连续、容易制造的最终轮廓。
- 肩面可能贡献较大面积和辐射收益，却没有相应的法向退蚀。
- 50 个宏步达到上限时仍可能返回 `status=OK`，缺少 `stepLimited` 硬判定。
- 当前最优 trial 68 的高能量排名会因外接球、肩面退蚀和电压策略改变。

### 7.3 Zigzag 簇特有问题

- `vol_tol=0.03` 不符合“体积严格相等”。
- 局部 block 温度已读取，但实际几何统一采用最小 `block_side`，局部状态没有落实到几何。
- 端部 stub 侧壁、转角面和 block 端面缺少一致的退蚀处理。
- C10 的单条横向 run 约 92 mm、总路径 1.118 m；题目未给硬包络限制，但机械稳定性、热变形、重力、加工和安装合理性不足。
- cap 案例可能漏积分最后时间区间。
- C10 `Tmin` 比固定端温低约 1 K，需排除数值/选择集错误。

### 7.4 优化和分析脚本问题

- 优化器默认调用 `max_safe`，没有调用 runner 已有的 `full_scan`。
- 单目标函数在 U 优于旧基线后不再奖励继续降低 U，无法主动冲击官方 `<10%` 档。
- 对寿命、能量和均匀性使用人为加权，缺少 Pareto 前沿和约束敏感性。
- Optuna sampler 没有显式固定 seed；resume 代码部分依赖私有存储 API。
- 多个路径硬编码；旧 `parse_results.py` 和 `surrogate_analysis.py` 默认读取初赛 `trials.csv` 和旧 baseline。
- 新物理版本重算前，任何 SHAP/代理模型结论都会学习错误的辐射和退蚀目标。

### 7.5 Java/COMSOL 展示模型问题

- `cylinder_process_views.java` 仍是均匀 2.5 mm 圆柱，不是当前圆柱候选。
- `zigzag_process_views.java` 仍是 N=8、L=104 mm 的旧结构，不是 C10。
- 力学使用常数 `E=411 GPa`、`nu=0.28`、`alpha=4.5e-6/K`，且两端完全固定；高温材料软化、轴向滑移和蠕变未评估。
- 四个稳态阶段只是数据集切换，不等于真实瞬态研究，也没有自动保存最终 MPH、STEP、STL 和动画。

## 10. 面向高分的改进方向

### 10.1 先拿稳物理和系统分

- 用外接球逃逸、能量守恒、收敛矩阵和材料敏感性证明结果可信，这比继续堆 trial 更能提高“仿真系统”和“物理洞察”分。
- 把每个候选的输入参数、几何、物理版本、失败状态、全过程历史自动归档，答辩时可从任一 CSV 行复现 MPH。
- 对圆柱 trial 68 与 zigzag C10 做“收益来源分解”：面积、温度、0-3 um 光谱占比、遮挡、端部导热、退蚀热点分别贡献多少。

### 10.2 将优化改为受约束多目标

- 主目标使用 `lifeTotalP03escape_J`，同时把寿命、初始功率、U、结构尺度和应力作为独立目标/约束，不再只靠单一手工加权分数。
- 先用低成本解析/代理模型筛选，再由 COMSOL 高保真复核，控制总迭代数并体现轻量化方法。
- 新搜索采用固定 seed、重复种子和 Pareto 前沿；最终选“稳健解”，不只选一次运行的最大值。

### 10.3 避免利用模型漏洞得到虚高方案

- 不允许存在只增加辐射面积却不参与退蚀的肩面、端面或 stub。
- Zigzag 虽可超出原圆柱包络，但应加入软包络、最小间距、最小弯曲半径、最大长细比和结构稳定性约束。
- 对 C10 这类超长路径至少比较：无重力/有重力、两端固定/一端滑移、室温/高温材料参数，避免最终方案只能在理想数学模型中成立。

### 10.4 争取创新和解释分

- 在物理闭环后，用代理模型的 SHAP/PDP 和参数扫描提炼“何处增材、何处减材、为何降低热点或遮挡”的规则。
- 可增加小规模 model-based RL 或主动学习作为外层选点器，但不应在物理模型修正前投入大量算力。
- 完成选做的表面起伏鲁棒性：同一候选在无起伏和 200 um/500 um 正弦起伏下比较寿命和热点迁移。

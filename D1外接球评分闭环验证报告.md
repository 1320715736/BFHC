## 1. 本轮修正

1. 修正 COMSOL 光谱分界：`lambda_r` 必须传数值 `3`，旧写法 `lam03=3[um]` 会被 API 再乘一次 `um`，导致 3 um 以上波段错误为零。
2. `P03gross` 改用 COMSOL 精确谱带发射量 `rad.epsilonu_band1*rad.ebu1`。
3. `P03escape` 改为所有 S2S 边界上的 `rad.J_band1*rad.Famb1` 积分，表示最终逃逸到黑色外部环境的功率。
4. 全波段分别使用 band 1 与 band 2 求和；0-3 um 仍是唯一评分波段，3 um 以上只参与热平衡。
5. 旧字段 `P03sphere`、`lifeTotalP03sphere_J` 保留为 escape 兼容别名，并强制写入 `metricVersion=radiation_escape_v2`。
6. 新增 gross、escape、自吸收、原始遮挡损失、非负报告损失、数值超额、环境污染和面积平均 `Famb` 字段。
7. 圆柱、zigzag 的 Python runner 与四个 Java 脚本已同步；旧 C 阶段 CSV schema 与 metric v2 不一致时禁止追加。
8. D1 控制 CSV 增加 schema 自动迁移；旧表头缺少 `erosionSolveRetries` 时按行长度无损对齐，禁止静默写出错位列。

## 2. 0 K 统计说明

- 热平衡仍使用 `293.15 K` 黑体环境，不把评分面错误地作为导热实体。
- 生产统计采用 S2S 环境逃逸 radiosity；同时输出 `initialP03ambient_W` 和 `initialAmbient03ToEscape_pct`。
- 四个稳态控制点的 293.15 K 环境在 0-3 um 内占 escape 的比例均小于 `5.2e-6%`，因此对 0 K 评分目标的相对误差可忽略。
- 已完成独立封闭腔 radiosity 基准：两个相互辐照灰体面外加 0 K、发射率 1 的黑色包络球，解析验证球面吸收功率严格等于 `J*Famb` 环境逃逸功率。
- 包络球半径取结构尺度的 `1.05`、`1.5`、`2.0` 倍时，上述积分在双精度容差内保持不变；因此生产模型无需增加会干扰热物理选择集的实体球。
